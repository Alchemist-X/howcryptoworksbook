## 一、量子计算基础：为什么它能威胁加密

### 经典计算 vs. 量子计算

经典计算机用**比特**（bit）——只能是 0 或 1。量子计算机用**量子比特**（qubit）——可以同时处于 0 和 1 的叠加态，直到被观测的瞬间才"坍缩"为确定值。

**一个直觉性比喻：**

```
破解加密 —— 经典计算机：
  在巨大的草垛中找一根针。逐根检查，可能需要数千年。

破解加密 —— 量子计算机：
  用磁铁吸出那根针。量子的并行探索能力 + 干涉效应
  让正确答案被放大，错误答案被抵消。

量子抗性加密 —— 换成铝针：
  磁铁不再有效，量子计算机又得回到逐根检查的老路。
```

### 哪些密码学安全，哪些危险

量子计算机**并非万能加速器**，只对特定类型的数学问题有颠覆性优势：

| 密码学类型 | 代表算法 | 量子威胁 | 原因 |
|-----------|---------|---------|------|
| 公钥加密 | RSA、ECC | **致命** | Shor 算法可利用数学结构直接反推私钥 |
| 对称加密 | AES-256 | 较小 | Grover 算法仅将安全强度减半，加长密钥即可 |
| 哈希函数 | SHA-256 | 较小 | 同上——使用更长输出即可保持安全 |

**核心洞察：** 量子抗性方案使用缺乏量子算法可利用的优雅结构的数学问题。这些替代问题即使对量子计算机也很难。

### 时间线：暴风雨何时到来？

```
当前状态（2025）               近期（~2030s早期）         中远期（2035-2040s+）
───────────────               ───────────────          ─────────────────
· Google "量子回声" 里程碑      · 最乐观的专家预测          · 多数专家共识的
· 能计算分子结构（经典无法）    · 仍远不足以威胁加密          破解加密窗口
· 距破解加密需要的规模          · Buterin 预测：可能在        · 可能更晚
  差几个数量级                   2028 前威胁以太坊           （或因 AI 突破而更早）

破解 RSA-2048 的估计需求：
  · 早期估计：~2000 万量子比特 + 8 小时
  · Gidney 最新估计：<100 万量子比特 + 不到一周
  · 前提：接近完美的量子计算机，几乎零错误
  · 当前技术远未达到
```

**"先窃取，后解密"风险：** 恶意行为者可能现在就在收集加密数据，等待量子计算机成熟后再解密。这使得保护长期秘密尤为紧迫。

### NIST 的密码学标准化

美国国家标准与技术研究院（NIST）花了近十年运行公开竞赛，审查和筛选量子抗性算法。2024 年 8 月，NIST 最终确定了首批三个标准：

| 方法类别 | 优势 | 用途 |
|---------|------|------|
| **格基密码学（Lattice-based）** | 效率高 | 加密 + 签名 |
| **哈希基签名（Hash-based）** | 安全假设简单、置信度高 | 数字签名 |

多样性提供了保险：如果一种数学方法被攻破，生态系统可以切换到替代方案。

## 二、区块链的特殊脆弱性

### 永久公开记录 = 永久攻击面

传统金融系统可以在幕后轮换加密密钥。但**区块链上发布过的每一个签名都是永久公开的**——一旦量子计算机成熟，这些历史数据就成为潜在的攻击面。

### 两种量子攻击的比喻

```
Shor 算法 = 万能锁匠
  从锁的外观（公钥）直接逆向工程出匹配钥匙（私钥）
  → 对当前区块链签名方案是灾难性的

Grover 算法 = 超人图书管理员
  仍需在书架中搜索，但效率大幅提升（安全强度减半）
  → 威胁较小，加长哈希即可防御
```

### 公钥暴露模型

**关键原理：量子计算机能破解公钥，但不能轻易破解公钥的哈希。**

把比特币地址想象成一个保险箱：密码（公钥）在有人打开它之前不会暴露。一旦打开，任何窃听者都能记录密码。今天的窃听者用不了这个密码——但等"量子撬锁工具"到来时，他们可以用记录的密码偷走箱内剩余的一切。

### 比特币：谁在危险中？

#### 高风险：P2PK 输出（2009-2012 早期）

```
P2PK 交易格式：
  "这是公钥，任何能证明控制权的人都可以花费这笔钱"
  → 公钥直接写在区块链上，零密码学保护

受影响规模：
  · 超过 150 万 BTC（比特币总供应量的 ~8.7%）
  · 包括中本聪的早期挖矿奖励
  · 当前估值超过 $1,000 亿
  → 相当于保险箱上直接贴着密码
```

#### 高风险：地址重用

早期用户常将同一地址用于多笔交易。每次花费都暴露公钥。如果地址上还有余额，这些"找零输出"就是量子攻击的现成目标。

#### 较安全：现代地址格式

| 格式 | 保护机制 | 量子抗性 |
|------|---------|---------|
| P2PKH / 原生 SegWit | 只存储公钥的哈希，花费时才暴露公钥 | **较强**（公钥隐藏） |
| Taproot（默认密钥路径） | 直接嵌入公钥 | **较弱**（与 P2PK 类似） |
| 单次使用地址 | 花费时暴露公钥，但已无余额 | **安全**（没有可偷的） |

#### 以太坊的暴露模式

以太坊的账户模型有不同的暴露模式：每笔交易都暴露可恢复的公钥。但**从未发送过交易的账户**仍受保护。一旦发送第一笔交易，公钥就永久暴露于该地址的所有未来存款。

智能合约钱包的认证逻辑存在于可升级代码中，**原则上**可以切换到量子抗性签名方案而不改变钱包地址。但目前在 EVM 上验证后量子签名的 Gas 成本过高，这条升级路径还基本是理论层面的。

### 休眠钱包与"量子抢夺"

休眠地址中暴露的公钥构成系统性风险：

```
                        "量子抢夺"场景

  强大的量子计算机突然出现
         ↓                              ↓
  恶意行为者竞相破解           网络开发者和社区竞相部署
  脆弱地址并窃取资金           紧急分叉以冻结/迁移资产
         ↓                              ↓
      谁先行动？ ←──── 博弈论的残酷动态 ────→ 谁先行动？

  超过 $1,000 亿的暴露价值 = 量子霸权的巨额悬赏
  → 国家和资金充足的私人实体有具体的经济激励加速量子计划
  → 竞赛不仅是谁建造计算机，更是谁能在窗口关闭前提取最大价值
```

## 三、各链的量子抗性过渡策略

### 比特币：技术方案与哲学矛盾

#### BIP-360 提案

引入专为量子抗性设计的新地址类型，基于 Taproot 架构但禁用暴露公钥的特性，替换为量子安全方案。渐进式升级，不破坏现有功能。

#### 争议焦点：强制迁移 vs. 自愿迁移

| 方案 | 内容 | 争议 |
|------|------|------|
| **Lopp 的 QBIP** | 多年弃用计划，激活约 5 年后设立"旗帜日"使脆弱消费失效 | 温和但耗时 |
| **Cruz 的 QRAMP** | 硬性截止日期强制升级 | 可能使休眠资金不可花费 |
| **承诺方案** | 当前持有者可证明所有权并安全转移 | 技术复杂度 |
| **有限制的认领** | 允许脆弱币被认领但限速，竞争推高费用流向矿工 | 道德争议 |

#### 休眠资产的三种处理方式

```
选项 A：永久销毁            选项 B：放任不管            选项 C：限速认领
──────────────            ──────────────            ──────────────
· 防止量子窃取             · 让量子能力者认领           · 允许认领但施加限制
· 违反"你的密钥=你的币"    · 视为"数字打捞"            · 竞争抬高费用给矿工
  的根本承诺               · 可能造成市场混乱           · 折中方案
· 社区强烈反对             · 不做任何保护              · 仍有争议
```

**深层矛盾：** 保护网络免受量子攻击，可能需要违反让比特币有价值的那些产权原则本身。中本聪曾在 2010 年讨论过面对密码学突破时采用新系统的必要性——但这只对仍控制私钥的人有效。

### 以太坊：工程权衡而非哲学冲突

以太坊更灵活的升级文化允许迭代解决方案：

**多管齐下的分阶段策略：**

- **EIP-7932：** 支持多种签名算法，在保持向后兼容的同时启用后量子方案
- **账户抽象（Account Abstraction）：** 作为关键入口，让智能钱包实现量子安全签名而无需立即修改协议
- **以太坊基金会资助：** 研究后量子多签方案

**签名大小的巨大挑战：**

```
当前以太坊签名：  65 字节

量子抗性替代方案：
  ├── ~2,400 字节 （37倍增长）
  └── ~29,000 字节（450倍增长）

直接影响：
  · 交易变大 → 存储需求和区块链膨胀
  · 更高的交易费用
  · 验证速度变慢 → 影响区块处理和网络吞吐量
```

**更深层的架构需求：** 数据可用性使用的密码学技术（如第二章讨论的 KZG 承诺）也需要量子抗性替代品。基于哈希和 STARK 风格的构造是有前途的候选方案，因为它们只面对 Grover 的温和加速，而非 Shor 的毁灭性优势。

以太坊基金会还有**紧急恢复分叉**提案——如果量子突破突然发生，可以快速冻结暴露账户。

### Solana：更紧迫的暴露问题

Solana 面临更即时的暴露风险：**大多数 Solana 账户地址在创建时就直接暴露公钥**，不像比特币或以太坊可以隐藏到交易发生。

```
比特币（现代格式）：地址 = hash(公钥) → 公钥隐藏到花费时
以太坊：          地址 = hash(公钥) → 公钥隐藏到首次发送交易
Solana：          地址 = 公钥本身   → 从创建之日起就暴露
```

2025 年 12 月，Solana 基金会与 Project Eleven 合作进行威胁评估并在测试网上原型化后量子数字签名——定位为前瞻性迁移演练而非紧急响应。

同时，Solana 生态系统已实验了基于哈希的一次性签名作为可选的钱包级保护。这作为权宜之计有用，但不是完整的全网迁移方案。

## 四、用户当前可以采取的最佳实践

### 比特币用户

```
✓ 避免地址重用——花费整个 UTXO 到新地址
✓ 确保没有价值残留在已暴露公钥的地址上
✓ 关注 BIP-360 等升级提案的进展
✗ 多签和 MPC 本身不消除量子风险（底层签名方案仍脆弱）
  但提供了后量子算法就绪时的升级路径
```

### 以太坊用户

```
✓ 避免在首次交易后的地址上保留大量资金
✓ 迁移到新的未使用地址，或最好是可升级的智能合约钱包
✓ 智能合约钱包 = 未来最佳升级路径
  （认证逻辑在可升级代码中，可换成后量子方案而不换地址）
✗ 需等待以太坊添加高效的内置后量子签名验证支持
```

### 所有区块链用户

> 个人密钥管理无法保护整个生态系统。数百万 BTC 位于暴露的遗留输出中，无数重用地址来自比特币早期——这些都需要**协议级别**的协调应对。问题不再是区块链是否需要协议变更，而是如何在不破坏现有功能或造成不可接受的经济冲击的情况下实施它们。

## 章节要点速览

1. **量子计算机通过 Shor 算法对公钥密码学（RSA、ECC）构成致命威胁，但对称加密和哈希函数只需加长密钥/输出即可安全**
2. **时间线不确定：最早 2030 年代初，更可能 2035-2040 年代——但"先窃取后解密"风险意味着现在就需要行动**
3. **区块链的独特脆弱性在于永久公开记录：链上发布的每个签名都是潜在的未来攻击面**
4. **比特币 P2PK 输出（~150 万 BTC，价值超 $1,000 亿）和地址重用是最高风险；现代哈希地址 + 单次使用显著更安全**
5. **量子抗性过渡的核心困难不是技术，而是社会协调：比特币的产权哲学 vs. 安全需求，以太坊的签名膨胀 37-450 倍，Solana 的全面公钥暴露**
6. **NIST 已于 2024 年最终确定首批量子抗性标准（格基 + 哈希基），为行业过渡提供基础**
7. **用户现在能做的：避免地址重用、将资金迁移到未暴露地址或可升级智能合约钱包、关注协议升级进展**
