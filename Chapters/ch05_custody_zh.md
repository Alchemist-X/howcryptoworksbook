## 一、范式转变：价值变成了信息

加密货币将价值从物理形态转化为纯粹的信息。不再需要运钞车和保险库——但一个新的现实诞生了：

> **谁控制密钥，谁就控制资产。**

一个 12 个单词的助记词可以承载数百万美元，没有任何物理痕迹。对于难民或生活在敌意政府下的人来说，这意味着价值可以在脑海中跨越国境、抵抗没收、绕过资本管制。

但这种能力伴随着对等的责任：交易不可逆，大多数损失源于操作失误而非密码学漏洞。

## 二、密码学基础

### 公钥、私钥与数字签名

```
私钥（256 位随机数）
  │
  │  单向数学运算（容易）
  ▼
公钥
  │
  │  哈希运算
  ▼
地址（公开标识符）

逆向推导？→ 以当前技术不可能
```

- **私钥**：通常从 12 或 24 个单词的助记词派生，而非直接生成
- **数字签名**：用私钥和交易详情创建——任何人都可以用公钥验证授权，但无法提取私钥
- **不可否认性**：签名后无法否认授权——数学提供了密码学证明

### 不同链的签名算法

| 链 | 算法 | 备注 |
|-----|------|------|
| Bitcoin | ECDSA + Schnorr (Taproot) | Schnorr 让多方签名看起来与单签一样 |
| Ethereum | ECDSA | |
| Solana | EdDSA | |

### 地址格式

| 链 | 格式 | 示例前缀 |
|-----|------|---------|
| Bitcoin | Legacy / P2SH / Bech32 | `1...` / `3...` / `bc1...` |
| Ethereum | 40 位十六进制 | `0x...` |
| Solana | 44 字符 Ed25519 | 直接表示公钥 |

### 助记词（BIP-39）

助记词将 128 位（12 词）或 256 位（24 词）熵编码为人类可读的单词序列。

关键属性：
- **确定性**：同一组词永远生成同样的密钥和地址
- **层次化**：一个种子可派生多条链的多个账户（BIP-32 派生方法 + BIP-44 路径标准）
- **可恢复**：仅凭助记词即可在不同软件中恢复整个钱包
- **第 25 个词**：可选密码短语——同一助记词生成完全不同的钱包，提供合理推诿性

> 高质量随机数生成至关重要。弱随机性 = 可预测的密钥 = 资产被盗。

## 三、个人自托管

### 软件钱包 vs 硬件钱包

| 维度 | 软件钱包 | 硬件钱包 |
|------|---------|---------|
| 代表 | MetaMask, Trust Wallet, Phantom | Ledger, Trezor |
| 体验 | 便捷，DeFi 集成好 | 需物理操作确认交易 |
| 安全性 | 继承宿主设备的所有漏洞 | 私钥永不离开安全芯片 |
| 适用 | 日常交易、小额资金 | 大额存储、长期持有 |

### 软件钱包的威胁图谱

```
外部攻击者
├── 恶意软件 / 键盘记录
├── 钓鱼网站（伪造钱包界面）
├── 供应链攻击（污染扩展/软件/硬件）
├── 剪贴板劫持（替换复制的地址）
├── 中间人攻击
└── 设备被盗 → 取证提取密钥
```

### 硬件钱包的安全模型

```
┌────────────────────────────┐
│     硬件钱包（安全芯片）      │
│  · 私钥在设备内生成和存储     │
│  · 交易在内部签名            │
│  · 只传输签名，不传输密钥     │
│  · 物理按钮确认每笔交易       │
└────────────────────────────┘
         │
    仅输出签名
         │
         ▼
  ┌──────────────┐
  │   主机电脑     │  ← 即使被入侵
  │  （可能有毒）   │     也无法提取密钥
  └──────────────┘
```

### 两种安全哲学

- **Ledger**：专有安全芯片 + 闭源固件 → 优先硬件级防篡改；3 次 PIN 错误即清除
- **Trezor**：全开源固件 → 社区可审计、可验证；错误 PIN 指数级延迟 + 可选自毁 PIN

两者都比软件存储有巨大安全优势。设备丢失不等于资金丢失——PIN 保护硬件，助记词备份支持在新设备上完全恢复。

### 操作最佳实践

1. **多地备份助记词**——防火灾/洪水摧毁单一备份
2. **定期恢复测试**——至少在第二台设备上做一次完整恢复
3. **保持固件更新**——修补新发现的漏洞
4. **逐字符验证地址**——防剪贴板劫持

> 当持仓超过 100 万美元、操作复杂度增加或出现组织需求时，个人托管模型达到极限——需要多方审批、合规能力和审计追踪。

## 四、机构托管模型

机构面临的根本问题与个人不同：**保护组织免受自身成员的威胁。**

### 新增威胁维度

| 威胁类别 | 说明 |
|---------|------|
| 内部人员风险 | 管理员滥权——恶意或失误；"就这一次"降级安全策略 |
| 操作失败 | 丢失密钥分片、未测试的灾难恢复、配置漂移 |
| 人为因素 | 单个管理员可能推翻技术控制——仍是最薄弱环节 |

### 四种主要托管模型

#### 1. 多签（Multisig）：透明标准

链上强制执行支出策略——公开、可审计。

- **优势**：利益相关者可验证治理决策和资金库操作；DeFi / DAO 首选
- **实现**：Bitcoin 原生多签 或 Ethereum Safe（原 Gnosis Safe）
- **代价**：交易更大（费用更高）、策略结构公开暴露决策过程
- **Taproot 升级**：Schnorr 签名让多方签名在链上看起来和单签一样，大幅降低透明性代价

#### 2. MPC 与门限签名：速度与隐私

多方联合执行密码学操作，**完整私钥从不存在于任何单一位置**。

```
参与者 A ─┐
参与者 B ─┼─→ 分布式签名协议 → 一个标准签名
参与者 C ─┘
          （无需重建完整密钥）
```

- **优势**：灵活变更签名者无需新地址；无额外链上开销；天然多链支持
- **风险**：依赖平台和供应商质量；部分广泛使用的协议已暴露私钥提取漏洞
- **代表供应商**：Fireblocks, Copper

#### 3. Shamir 秘密共享（SSS）：存储与恢复

将已有私钥拆分为 N 份，任意 K 份可重建（如 5 取 3）。

- **与 MPC 的关键区别**：签名前必须重建完整密钥 → 产生临时单点故障
- **更适合**：备份和恢复场景，而非频繁签名操作

#### 4. 合格托管人：监管框架

受监管的银行和信托公司提供法律隔离、监管审查和保险覆盖。

- **基础设施**：HSM（硬件安全模块）——企业级安全芯片 + 安全飞地 + MPC 纵深防御
- **温度分层**：热钱包 < 5% / 温钱包 5-10% / 冷存储 >= 90%
- **提款摩擦**：多日验证期、多渠道认证——刻意的缓慢是安全层
- **核心优势**：破产隔离（客户资产与托管人自有资产法律分离）

### 主要机构托管人

| 机构 | 牌照 | 特点 |
|------|------|------|
| **Coinbase Custody** | 纽约有限目的信托 | 隔离冷存储，审查员监督 |
| **Anchorage Digital** | 联邦特许银行 | "主动托管"——HSM + 安全飞地 + 生物识别 |
| **BitGo** | SD/NY 信托 | 链上多签 + 门限签名方案 |

### 交易所托管

即使有自托管或合格托管人关系，活跃交易仍需在交易所维持余额。

- **储备证明（PoR）**：链上/审计师验证的偿付能力证明——有用但不充分（时间点快照，可能遗漏链下负债）
- **最佳实践**：Kraken 的 Merkle 树负债证明 + 每客户包含性证明

## 五、历史教训：托管是怎么崩溃的

| 事件 | 年份 | 根本原因 | 损失 |
|------|------|---------|------|
| **Mt. Gox** | 2014 | 热/冷隔离模糊 + 无对账 | ~65 万 BTC |
| **Parity 多签** | 2017 | 共享库依赖——单个 Bug 影响数百实体 | ~51.3 万 ETH 冻结 |
| **Ronin 桥** | 2022 | 9 验证者多签中 5 个被控制 + 6 天无人发现 | ~6.15 亿美元 |
| **FTX** | 2022 | 挪用客户资产进行自营交易——无隔离 | 数十亿美元 |

### 共同教训

```
技术安全 ≠ 运营安全

· Mt. Gox：技术再差也不如每日对账缺失致命
· Parity：合约审计不是可选的奢侈品
· Ronin："去中心化"系统可因运营捷径变得中心化
· FTX：无论技术多先进，挪用客户资金 = 系统性风险

→ 需要严格流程、独立监督、隔离原则的始终执行
```

## 章节要点速览

1. **密钥等于控制**——加密货币将价值转化为信息，控制私钥就是控制资产
2. **助记词是可恢复性的基石**——一组 12/24 词可在任何兼容软件中恢复全部钱包
3. **硬件钱包是个人托管的黄金标准**——私钥永不暴露给可能被入侵的计算机
4. **机构托管的核心问题是保护组织免受自身的威胁**——内部人员风险和操作失败比外部攻击更常见
5. **多签提供透明，MPC 提供灵活**——不同模型适合不同场景，可以组合使用
6. **合格托管人提供技术无法替代的法律保护**——破产隔离、监管审查、保险覆盖
7. **历史上最大的托管失败都不是密码学被攻破**——而是操作失误、隔离缺失和人为因素
