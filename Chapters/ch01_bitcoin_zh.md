## 一、起源与核心理念

2009 年 1 月 3 日，匿名创建者中本聪在比特币创世区块中嵌入了一条《泰晤士报》的标题："Chancellor on brink of second bailout for banks"（财政大臣即将对银行进行第二轮紧急救助）。这条信息是对中心化金融系统失败的永久性批判，也是比特币的设计宣言。

比特币的核心特性：

- **点对点电子现金系统**——无需可信中介
- **透明、可预测的货币政策**——由数学而非央行执行
- **可独立验证的有限供应**——人类历史上唯一可以被任何人独立验证总量上限的资产

## 二、挖矿与工作量证明

### 哈希函数与挖矿过程

矿工将交易打包成区块，然后反复运行 SHA-256 哈希算法尝试找到低于网络目标阈值的输出值。

```
挖矿过程简化：

区块数据 + Nonce (0, 1, 2, ... 40亿)
         │
         ▼
    SHA-256(SHA-256(...))
         │
         ▼
    输出哈希值 < 目标阈值？
         │
    ┌────┴────┐
    否        是
    │         │
 换 Nonce   成功！广播区块
 或修改      获得区块奖励
 coinbase
 交易
```

关键概念：

| 术语 | 含义 |
|------|------|
| **Nonce** | 矿工不断递增的计数器（0 到约 40 亿），每次改变产生不同哈希 |
| **Coinbase 交易** | 每个区块中的特殊交易，向矿工支付区块奖励 + 手续费（"创造"新比特币） |
| **哈希率** | 矿工每秒尝试的哈希次数，衡量算力 |
| **难度调整** | 每 2,016 个区块（约两周）自动调整，保持平均出块时间 10 分钟 |
| **ASIC** | 专为 SHA-256 挖矿设计的芯片，效率是普通电脑的数千倍 |
| **矿池** | 多个矿工联合算力、按贡献比例分配奖励，平滑收入波动 |

新创造的币需要等 100 个确认区块（约 16-17 小时）才能花费，防止矿工通过重写历史获利。

## 三、共识机制：中本聪共识

比特币如何在没有中央权威的情况下就"谁拥有什么"达成一致？

### 节点与最长链规则

**全节点**独立验证每一笔交易和区块，维护区块链的完整副本。中本聪共识的核心不是"最长链"，而是**累计工作量最大的链**。

```
分叉示例：

    ┌── 区块 A'（1000 次简单计算）── 区块 B'
    │
主链 ─┤
    │
    └── 区块 A（600 次高难度计算）── 区块 B ← 累计工作量更大，胜出
```

### 临时分叉与确认

两个矿工几乎同时找到有效区块时会产生临时分叉。下一个区块找到后，较长的链自动胜出，短链上的区块作废。这就是为什么商家通常等待多个确认（大额交易等 6 个确认）再视为最终交易。

## 四、货币政策

比特币拥有算法化的、完全可预测的货币政策：

```
区块奖励减半时间线：

50 BTC ──→ 25 BTC ──→ 12.5 BTC ──→ 6.25 BTC ──→ 3.125 BTC ──→ ...
2009       2012       2016        2020        2024

每 210,000 个区块（约 4 年）减半一次
约 2140 年补贴降为零，总供应收敛于 ~20,999,999.9769 BTC
截至 2026 年初，约 95% 已被挖出
```

**安全预算 = 区块补贴 + 交易手续费。** 2024 年，区块补贴占矿工总收入约 94%。随着补贴持续减半，交易手续费能否支撑网络安全是比特币面临的核心长期挑战。

## 五、技术架构：UTXO 模型

比特币不使用"账户余额"模型，而是使用 **UTXO（未花费交易输出）** 模型。最好的类比是现金：

```
你的钱包里有：$20 + $5 + $5 + $1 + $1 = $32

买 $7 的东西：
  输入：$5 + $1 + $1 = $7（这三张钞票被消耗）
  输出：$7 给商家（新 UTXO）

如果你用 $20 买 $7 的东西：
  输入：$20（消耗）
  输出：$7 给商家 + $13 找零回自己（两个新 UTXO）
```

### 密钥与地址

```
私钥 ──(椭圆曲线加密)──→ 公钥 ──(哈希)──→ 地址

· 私钥：约 2^256 种可能，核心秘密
· 公钥：数学上与私钥关联，但不可逆推
· 地址：公钥的哈希，更短更易分享
· 花费时才需暴露公钥（地址隐藏公钥直到花费时刻）
```

### 地址格式演进

| 格式 | 前缀 | 特点 |
|------|------|------|
| Legacy | `1` | 最老格式，兼容性最好，费用较高 |
| P2SH | `3` | 常用于多签，向后兼容 |
| Native SegWit | `bc1q` | 现代默认，费用低，全小写 |
| Taproot | `bc1p` | 最新格式，直接编码公钥，更灵活的隐私条件 |

**建议：使用钱包默认生成的地址格式（通常是 SegWit 或 Taproot）。**

## 六、交易与手续费市场

交易从用户广播后进入节点的**内存池（mempool）**，等待矿工选取打包。区块空间有限，矿工按**手续费率（sats/vB）** 优先选择交易。

```
手续费率 = 手续费 / 交易大小（虚拟字节）

小交易付 10 sats 可能比大交易付 100 sats 有更高的费率
→ 矿工优先打包费率高的交易
```

交易卡住时的应对：
- **RBF（费用替换）**：发送方广播更高费用的替代交易
- **CPFP（子替父）**：创建一笔高费用子交易，激励矿工连同父交易一起打包

---

## 七、隐私模型

比特币是**假名制**，不是匿名制。地址不直接关联真实身份，但链上分析可以聚类地址并追踪资金流。Chainalysis 等公司已基于去匿名化构建了价值十亿美元的业务。

**最佳实践：**
- 每笔交易使用新地址
- 避免合并来自不同来源的 UTXO
- 考虑使用 **CoinJoin**：多用户的输入混合成一笔交易，产生等额输出，打破"多输入属于同一人"的启发式链接

---

## 八、升级机制：去中心化网络如何演进

### 分叉类型

| 类型 | 兼容性 | 风险 | 实例 |
|------|--------|------|------|
| **硬分叉** | 不兼容，所有人必须升级 | 永久分裂网络 | Bitcoin Cash（2017） |
| **软分叉** | 向后兼容，收紧规则 | 需多数支持 | SegWit、Taproot |

### 关键升级

**SegWit（2017）：** 解决了交易延展性 bug，将签名数据移至单独的"见证"结构，引入区块权重（4M 权重单位上限），为闪电网络铺平道路。其激活过程堪称比特币治理的经典案例——矿工用信号位阻挠升级，BIP 148（UASF）的经济威胁最终迫使矿工让步。

**Taproot（2021）：** 引入 Schnorr 签名（支持密钥和签名聚合）和 MAST（默克尔化抽象语法树），使复杂交易在链上看起来与简单支付无异，大幅提升隐私性和可扩展性。

### 协议骨化

比特币社区中许多人倡导**协议骨化**——随着比特币市值增长，变更的风险阈值应该越来越高。比特币的力量部分来自于它**不做什么**。

## 九、安全模型

### 51% 攻击

控制多数算力的实体可以尝试重写近期历史或双花。但：

- **逐利攻击者**：获取和运行足够硬件的成本巨大，成功攻击会使他们正在攻击的资产贬值
- **国家级攻击者**：面临实际操作障碍，且有更廉价有效的工具（监管、税收、向交易所施压）

每个后续区块使篡改一笔交易所需的工作量呈指数增长。比特币被设计为**反脆弱**的——它从压力和攻击中变得更强。

## 十、Layer 2 扩展

### 闪电网络

比特币基础层为高保证结算优化——小额日常支付不经济。闪电网络将频繁的小额支付转移到链下。

```
支付通道运作：

Alice ←─── 链上锁定资金 ───→ Bob
         (需双方签名)
           │
           ▼
      链下更新余额（无限次）
      作弊 → 加密惩罚机制
           │
           ▼
      关闭通道 → 最终余额上链结算

路由：Alice → Bob → Carol（Alice 不需要和 Carol 有直接通道）
```

**优势：** 秒级结算、极低费用、优于重复使用链上地址的隐私性。

**核心挑战——流动性：** 发送需要出方向容量，接收需要入方向容量。通道流动性不足导致支付失败。

**托管权衡：** 复杂性驱使许多用户转向托管或半托管闪电钱包——用户体验改善了，但重新引入了比特币原本要消除的信任需求。

### Ordinals 与 BRC-20

比特币作为数据层的新用途：

- **Ordinal 理论**：为每个聪分配序号，使其可追踪、可附加数据（铭文）
- **铭文**：将任意数据（图像、文本等）直接嵌入区块链的见证数据中
- **BRC-20**：通过 JSON 铭文在比特币上创建同质化代币（依赖链下索引器解释）

争议的核心：这究竟是对区块空间的"滥用"，还是为矿工长期可持续性提供关键手续费收入的合法用途？

---

## 十一、L2 信任模型与契约提案

比特币脚本无法验证外部状态或强制约束未来交易。当前大多数"L2"实际上是侧链或联邦网络——依赖第三方验证器，而非比特币自身共识。

**潜在解决方案：**

| 方案 | 原理 | 状态 |
|------|------|------|
| **契约（Covenants）** | 新操作码（OP_CAT、CTV、CSFS）让比特币脚本可验证 L2 证明 | 提案阶段，需社区共识 |
| **BitVM** | 乐观验证模型——操作者提交声明，任何人可在窗口期内挑战 | 实验性，已发现安全漏洞 |

## 章节要点速览

1. **比特币是一个无需信任中介的点对点电子现金系统，由中本聪共识保障安全**
2. **工作量证明通过难度调整保持约 10 分钟出块，ASIC 和矿池主导挖矿格局**
3. **UTXO 模型类似现金——花费时消耗整个"钞票"并创建新输出**
4. **私钥 → 公钥 → 地址的单向推导链是所有权的密码学基础**
5. **每 4 年减半的货币政策创造可预测的稀缺性，但长期安全预算是开放问题**
6. **SegWit 和 Taproot 是两次最重要的升级，分别解决了延展性和隐私/可编程性问题**
7. **闪电网络解决了小额支付问题，但流动性管理和操作复杂度限制了其采用**
8. **Ordinals 和 BRC-20 将比特币从"支付轨道"扩展为"永久存储层"，争议持续**
9. **真正的 L2 安全需要契约级别的升级——当前方案依赖信任假设**
