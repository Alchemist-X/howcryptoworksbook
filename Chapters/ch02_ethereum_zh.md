## 一、核心理念：从"数字黄金"到"世界计算机"

比特币实现了去中心化的数字稀缺性。以太坊在此基础上更进一步——让计算本身变得可编程且去中心化。

这一转变解锁了全新的可能性：去中心化交易所让人们无需中介即可交易代币；借贷协议让用户通过**智能合约**赚取利息或借款；NFT 市场创造了新形式的数字所有权。更关键的是，这些应用可以**无缝协作**——借贷协议可以自动与交易所交互，金融产品从平台本身有机涌现。

```
比特币 vs 以太坊的设计哲学：

比特币：简单性 + 安全性优先
        → 受限的脚本语言，专注价值转移

以太坊：可编程性 + 表达力优先
        → 图灵完备的虚拟机，通用计算平台
        → 更高的复杂性 = 更大的攻击面
```

## 二、以太坊虚拟机（EVM）

EVM 是以太坊的核心计算引擎，在数千台计算机（节点）上同时执行代码。

### 运作原理

EVM 是基于栈的虚拟机，使用底层指令（操作码）处理计算。开发者用 Solidity 或 Vyper 等高级语言编写智能合约，编译器将其翻译成 EVM 字节码。每个节点独立执行相同计算并验证最终状态一致——这就是**去中心化共识**。

每个操作消耗 **Gas**（以计算工作量衡量的费用），目的是：
1. 补偿节点运营者的计算成本
2. 防止垃圾交易（每个操作都有成本）
3. 确保没有交易可以无限运行

### EVM 的网络效应

EVM 已成为行业事实标准。大多数 Rollup（Arbitrum、Optimism、Base）和许多替代 L1 都采用了 EVM 兼容性：

```
EVM 兼容的价值飞轮：

应用（Uniswap、Aave）可最小改动部署到新链
       │
       ▼
整套基础设施（MetaMask、区块浏览器、开发工具）跨链通用
       │
       ▼
新链可继承以太坊成熟的工具链和用户群
       │
       ▼
网络效应强化以太坊的主导地位
```

### 可扩展性瓶颈

由于每个全节点必须重放每笔交易，以太坊本质上是一台全球复制的计算机。Gas 上限和出块时间必须足够保守，让普通机器能跟上。这就是为什么原始吞吐量根本性稀缺——扩容必须通过分层和更高效地使用这一稀缺资源来实现。

## 三、手续费系统：EIP-1559

### 旧模式 vs 新模式

| 维度 | EIP-1559 之前 | EIP-1559 之后（2021.8） |
|------|-------------|----------------------|
| 定价方式 | 混乱的拍卖——用户盲猜出价 | 算法基础费 + 用户小费 |
| 可预测性 | 差——出价过高或过低 | 好——钱包可准确估算 |
| 费用去向 | 全部给矿工 | 基础费**燃烧**，仅小费给验证者 |

### 关键机制

```
用户设置：
  maxFeePerGas = 愿意支付的最高价格
  maxPriorityFeePerGas = 给验证者的小费

实际支付 = min(maxFeePerGas, baseFee + priorityFee)
总交易成本 = 使用的 Gas × 有效 Gas 价格

基础费调节规则：
  区块使用 > 目标（Gas 上限的一半）→ 基础费上调最多 12.5%
  区块使用 < 目标                  → 基础费下调最多 12.5%
```

### 通缩机制

基础费被永久燃烧（销毁），从流通中移除。当持续高需求使燃烧速率超过质押奖励的新增发行量时，ETH 供应量净减少（通缩）。

自 2022 年合并以来，ETH 经历了多段通缩期。但 Dencun 升级后 L1 Gas 变便宜，燃烧减少，2024 年后 ETH 重新出现净通胀期。

## 四、账户模型与代币标准

### 账户类型

| 类型 | 控制方式 | 用途 |
|------|---------|------|
| **EOA（外部拥有账户）** | 私钥 | 普通用户钱包 |
| **智能合约账户** | 代码逻辑 | 可编程账户，触发时执行代码 |

以太坊地址是公钥哈希的最后 20 字节（如 `0x742d35Cc...`）。**ENS**（以太坊域名服务）允许注册人类可读名称（如 `alice.eth`）来映射这些地址。

**EIP-7702（Pectra 升级）** 开始模糊 EOA 和智能合约的边界——允许 EOA 临时委托控制权给智能合约代码，实现赞助交易、批量操作等功能。

### ERC-20：统一的代币标准

ERC-20 之前，每个代币都是独一无二的"雪花"。ERC-20 建立了通用接口（`transfer()`、`approve()`、`balanceOf()`），引爆了 DeFi 的"寒武纪大爆发"：

> 开发者可以构建适用于数千种代币的应用，无需为每种写定制代码。DEX 可以上架任何 ERC-20 代币，借贷协议可以接受任何 ERC-20 作为抵押品。

这种**可组合性**（协议像乐高积木一样协作）是以太坊最核心的特征之一。

## 五、共识机制：从挖矿到质押

### 合并（The Merge）：2022.9.15

以太坊从能源密集型的工作量证明转向权益证明——能耗降低超过 99.9%，同时维持安全保障。

### 权益证明的时间节奏

```
12 秒 = 1 个 Slot（时隙）
32 个 Slot = 1 个 Epoch（纪元）≈ 6.4 分钟

每个 Slot：
  · 1 个验证者被随机选中提议区块
  · 数百个验证者提供证明（attestation）= 加密投票

最终性（Finality）路径：
  区块获 2/3 验证者证明 → "合理化（justified）"
  下一纪元再获 2/3 确认 → "最终化（finalized）"
  ≈ 12.8 分钟
```

### 质押经济

- 最低激活阈值：32 ETH
- Pectra 升级后（EIP-7251）：单个验证者有效余额可扩展至 2048 ETH
- **BLS 签名**：将数千个签名压缩为单个紧凑证明
- **罚没（Slashing）**：提议冲突区块或矛盾证明 → 基础罚金较小，但关联攻击罚金随参与者数量急剧放大

## 六、流动性质押

质押者面临两难：锁定代币保障网络安全赚取奖励，还是保持流动性参与 DeFi？

**流动性质押协议**通过发行可交易的 **LST（流动性质押代币）** 解决这一问题。

| 协议 | 市场份额 | 代币机制 | 特点 |
|------|---------|---------|------|
| **Lido** | ~85%（2026 初） | stETH：余额每日自动增长（rebase） | 专业节点运营商策展制 |
| **Rocket Pool** | ~5% | rETH：余额不变，兑换率上升 | 去中心化——允许独立运营者以最低 8 ETH 创建验证者 |

**风险：** Lido 的主导地位威胁网络去中心化；智能合约漏洞；验证者惩罚影响全池；市场压力下 LST 可能脱锚（2022 年 stETH 折价事件）。

## 七、Rollup：扩容革命

### 为什么不能简单"加大区块"

增大 Gas 上限或缩短出块时间 → 提高硬件要求 → 排挤小节点 → 削弱去中心化。以太坊核心开发者坚持让普通消费级硬件也能运行节点。

### Rollup 原理

```
┌──────────────────────────────────────────────┐
│              Layer 2（Rollup）                 │
│  · 交易在 L2 链上执行（更快、更便宜）           │
│  · 不需要每个以太坊节点重新执行                  │
├──────────────────────────────────────────────┤
│              Layer 1（以太坊）                  │
│  · 接收压缩的交易数据                           │
│  · 提供数据可用性、争议解决、最终结算             │
│  · 多用户分摊单笔 L1 交易的 Gas 成本            │
└──────────────────────────────────────────────┘
```

### 两种 Rollup 模式

| 维度 | 乐观 Rollup | ZK Rollup |
|------|------------|-----------|
| 代表项目 | Arbitrum、Optimism、Base | Starknet、zkSync、Scroll |
| 安全假设 | 先假设有效，7 天挑战期 | 数学证明每批交易正确 |
| 提款速度 | ~7 天（可用第三方快速桥） | 分钟到小时（证明生成+验证） |
| 成本 | 较低复杂度 | 较高计算成本（生成证明） |
| 证明系统 | 欺诈证明 | SNARK（小证明、需可信设置）或 STARK（大证明、无可信设置） |

### 排序器中心化问题

大多数 Rollup 目前依赖中心化排序器（Sequencer）来实现快速确认。正在开发的解决方案：共享排序网络、排序器轮换、包含列表（Inclusion lists）、预确认（Preconfirmations）。

## 八、数据可用性：Blob 与 EIP-4844

Rollup 的主要成本来自将数据发布到以太坊。2024 年 3 月的 Dencun 升级引入了 **Blob 交易**：

```
旧模式：Rollup 数据永久存储在执行层（昂贵）
        → 数据可用性成本占 Rollup 费用的 80-95%

新模式（EIP-4844）：
        Blob = ~128 KB 的数据包
        · 临时存储在共识层（约 18 天后自动删除）
        · 独立的费用市场（比普通 Gas 便宜得多）
        · KZG 承诺 = 加密指纹，唯一标识 Blob 内容

效果：Rollup 成本大幅下降
```

**替代数据可用性方案：**
- **Celestia**：专门的数据可用性链，使用数据可用性采样（DAS）
- **EigenDA**：利用以太坊再质押生态提供高吞吐数据可用性
- **Validium**：数据存在链下委员会管理，更便宜但信任假设更强

## 九、再质押（Restaking）

### EigenLayer 的核心机制

权益证明使质押资本的复用成为可能。验证者可以选择同时为多个**主动验证服务（AVS）** 提供安全保障：

```
ETH 质押者
    │
    ├── 原生再质押：将提款凭证指向 EigenPod
    │
    └── 流动性再质押：将 LST 存入 EigenLayer 策略
         │
         ▼
    为 AVS 提供共享安全
    （数据可用性、预言机、跨链桥、排序器...）
         │
         ▼
    违规 → 额外罚没（在以太坊级别惩罚之上）
```

### 技术架构要点

- **策略合约**：管理 ERC-20 存取款
- **EigenPod**：管理原生质押的验证者提款凭证
- **罚没合约**：每个 AVS 独立执行其特定规则
- **委托机制**：用户可委托给专业运营商
- **否决委员会**：防止错误或仓促的罚没执行
- **主体间罚没**：某些违规无法纯链上证明，依赖社会共识投票

### 当前现实与风险

| 维度 | 现状 |
|------|------|
| **收益来源** | 大部分来自激励计划和空投，非可持续费用收入 |
| **核心风险** | 关联罚没（一个错误触发多个 AVS 罚款）、智能合约漏洞叠加 |
| **提款延迟** | 以太坊退出队列 + EigenLayer ~14-17 天托管期 + AVS 额外限制 |
| **流动性风险** | LRT 代币可能脱锚，引发级联清算 |

> 再质押目前更像是对 EigenLayer 生态未来成功的杠杆押注，而非成熟的现金流资产类别。额外的负债已经存在，而补偿这些风险的费用市场仍在设计和实战检验中。

## 章节要点速览

1. **EVM 是以太坊的核心——图灵完备的虚拟机，已成为行业事实标准**
2. **EIP-1559 用算法基础费 + 燃烧机制取代了混乱的手续费拍卖，让 ETH 具备通缩潜力**
3. **账户模型（非 UTXO）支持复杂状态管理，ERC-20 标准引爆了 DeFi 可组合性**
4. **2022 年合并将以太坊从工作量证明转向权益证明，能耗降低 99.9%+**
5. **流动性质押解决了质押资本的流动性困境，但 Lido 的主导地位构成中心化风险**
6. **Rollup 是以太坊的扩容路线——乐观 Rollup 靠欺诈证明，ZK Rollup 靠有效性证明**
7. **EIP-4844 的 Blob 交易将 Rollup 数据可用性成本降低了一个数量级**
8. **再质押通过资本复用提供共享安全，但叠加了多层智能合约风险和关联罚没风险**
