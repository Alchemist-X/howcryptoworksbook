## 一、崛起之路：从无名项目到永续合约 DEX 霸主

### 增长轨迹

Hyperliquid 的崛起是 DeFi 历史上最戏剧性的竞争逆转之一：

```
2023 年：   交易量几乎为零
2024.08：   月交易量超过 dYdX（此前的绝对霸主）
2024.11：   HYPE 代币空投——31% 供应分配给社区，零 VC 分配
2025 年下半年：月交易量持续超 $2,000 亿，约为 Binance 永续的 15%
            吸引约 $60 亿桥接 USDC 抵押品
```

HYPE 代币从 $2 起步涨至近 $60（30 倍），背后的核心驱动力：**约 99% 的交易费流入 HYPE 回购**——将治理代币转变为对协议现金流的索赔权。

### dYdX 为何崩塌

dYdX 的市场份额从 2023 年 1 月的 75% 跌至 2024 年底的 7%。原因是**代币经济学和执行时机的双重失败**：

| 维度 | dYdX 的做法 | Hyperliquid 的做法 |
|------|-----------|------------------|
| 费用归属 | v3 费用归 dYdX LLC；v4 流向验证者/质押者（USDC），不产生代币买压 | ~99% 费用 → HYPE 回购，直接推动代币价值 |
| 回购力度 | 2025.03 才启动，仅捕获 25% 费用，质押而非燃烧 | 激进的费用-回购机制，从第一天起 |
| 迁移 | v4 迁移引入复杂桥接 + ~1s 延迟，在性能最关键时分散资源 | 从一开始就为交易场景优化 |
| 费用折扣 | 依赖质押要求 | 基于交易量和推荐层级，降低交易者摩擦 |

## 二、技术架构：HyperBFT 与 HyperEVM

### HyperCore：专为交易打造的 L1

Hyperliquid 构建了 **HyperCore**——一条专注性能的定制 L1 区块链。

**共识层：HyperBFT**
- 从 Tendermint 迁移至自研的拜占庭容错共识
- 只要超过 2/3 验证者诚实即保证终局性
- 中位交易终局时间 **0.2 秒**
- Epoch 约 100,000 轮（约 90 分钟）

**验证者经济学：**
- 自委托至少 10,000 HYPE 才能成为活跃验证者
- 佣金提升被严格限制在 1% 以下（防止先低佣金吸引委托再大幅涨价）
- 1 天委托锁定期 + 7 天解质押期

### HyperEVM：兼容以太坊的执行层

HyperEVM 提供完整的 EVM 兼容性，以 HYPE 为 Gas 代币，让现有以太坊钱包、工具和开发者工作流无缝接入。

#### 双区块架构

```
┌─────────────────────────────────────────────────────┐
│                   HyperBFT 共识                      │
├──────────────────────┬──────────────────────────────┤
│     HyperCore        │        HyperEVM              │
│  · 亚秒终局性         │  · 快车道：每秒出块（2M gas） │
│  · ~200k 订单/秒     │  · 慢车道：每分钟出块（30M gas）│
│  · 订单簿操作         │  · 智能合约部署和复杂交互     │
│                      │                              │
│  ← EVM 合约可读取     │  EVM 合约可写入 →            │
│    Core 状态          │    Core 操作队列             │
└──────────────────────┴──────────────────────────────┘
```

核心创新在于**双层通信机制**：
- **读取路径（同步）：** EVM 合约可直接访问 HyperCore 的永续仓位、现货余额、金库权益、质押数据、预言机价格
- **写入路径（异步）：** EVM 合约将操作入队（下单、管理仓位、转移抵押品等），在后续 Core 区块中执行

**资产链接（Asset Linking）：** 每个 HyperCore 现货资产可通过 `0x200...` 地址的桥接合约与 HyperEVM 上的 ERC-20 链接。链接后，资产在 Core 和 EVM 之间自由流动，无需包装代币或单独桥接。

#### Builder Codes：激励第三方前端

```
传统交易所：平台垄断界面 → 捕获所有费用
Hyperliquid：开放 Builder Codes → 第三方开发者赚取额外费用
  · 永续交易最高 0.1%，现货交易最高 1%
  · Phantom Wallet：6 个月 ~$1,000 万收入
  · Builder Code 集成已带来 >$1,000 亿永续交易量
  · 开发者总计赚取近 $4,000 万
```

效果：流动性不碎片化，但界面生态多样化。第三方前端已提供主界面所没有的功能（如私有 TWAP、高级订单类型）。

#### 抵押品系统

USDC 作为 Hyperliquid 的统一抵押品。2025 年 9 月，Circle 宣布将在 Hyperliquid 上发行原生 USDC 并投资 HYPE——这将逐步减少对 Arbitrum 桥接（3-of-4 验证者多签）的依赖。

## 三、可交易产品

Hyperliquid 提供三类交易产品，均运行在完全链上的订单簿上：

| 产品 | 定价来源 | 上架方式 | 用途 |
|------|---------|---------|------|
| **永续合约（Perps）** | 外部预言机 | 策展制 / HIP-3 无许可部署 | 标准杠杆交易 |
| **Hyperps** | 内部定价（无外部预言机） | 策展制 | 代币上线前的预启动交易/对冲 |
| **现货交易** | 链上订单簿 | HIP-1 荷兰拍卖获取 Ticker | 原生代币和桥接资产 |

### 桥接与资产表示

所有现货资产以 HIP-1 代币形式在 HyperCore 上交易：

```
BTC/SOL 等原生资产
  → Unit 的 lock-and-mint 守护系统（2-of-3 法定人数：Unit + Hyperliquid + Infinite Field）
  → 铸造 HIP-1 代币（如 UBTC）在链上订单簿交易
  → 提款时反向操作

USDC（来自 Arbitrum）
  → Hyperliquid 自有验证者桥接（4 验证者集，3-of-4 签名）
```

替代方案已出现：LayerZero 将 HyperEVM 作为一等端点，支持从 120+ 链转移代币。

### Hyperps 的风险

2025 年 8 月，四个协调的大户在 XPL hyperps 上执行操纵：仅用 $18.4 万将 XPL 现货价格推高近 8 倍，导致期货价格从 $0.60 飙升至 $1.80，引发超 $2,000 万的空头清算。获利约 $1,500 万。Hyperliquid 随后实施 10 倍价格上限等紧急保护措施。

## 四、HLP：社区驱动的做市与清算

**Hyperliquidity Provider（HLP）** 是 Hyperliquid 最具创新性的设计——一个社区所有的金库，同时承担做市和清算两大职能。

```
存款者 → 注入资金到 HLP
          ↓
     ┌────┴────┐
     │ 做市收益 │  · 买卖价差利润
     │         │  · 占日交易量约 1-2%（有机做市商已增长）
     ├─────────┤
     │ 清算收益 │  · 接管被清算仓位 + 抵押品
     │         │  · 在价格回弹时平仓获利
     └────┬────┘
          ↓
     利润（和亏损）按份额分配给存款者
```

### JELLY 操纵事件（2025.03）

攻击者开设大额杠杆仓位（$450 万做空 + 两笔 $250 万做多），操纵清算过程的同时在 Solana 上将 JELLY 价格拉高 250%。造成 $1,200 万未实现亏损，威胁协议偿付能力。验证者紧急介入，覆写预言机价格以防止崩溃。所有交易者获得补偿，但暴露了风险管理的重大漏洞。

### 十月清算级联（终极压力测试）

```
全市场：>$200 亿杠杆仓位被清算，未平仓合约暴跌 43%
Hyperliquid：处理了约一半的清算量
  · 未平仓从 $140 亿 → $60 亿
  · 100% 正常运行时间，零坏账
  · HLP 盈利 >$4,000 万（金库资本约 10% 日回报）
  · 首次触发 auto-deleveraging 机制（两年来首次）

对比 Lighter（竞争对手）：
  · LLP 金库遭受 ~5.3% 回撤（~$2,150 万亏损）
  · 基础设施出现数小时中断
  · 无法管理仓位的交易者额外损失 $2,500-3,000 万
```

HLP 的子金库架构和风险隔离机制在极端压力下证明了其设计的韧性。

## 五、去中心化挑战

Hyperliquid 的高性能是以有意识的中心化选择为代价的：

| 问题 | 具体表现 |
|------|---------|
| **验证者控制** | Hyper Foundation 通过自有验证者控制约 80% 的 HYPE 质押 |
| **闭源软件** | 节点软件不开源，验证者抱怨"盲签"场景，频繁被 jail |
| **API 依赖** | 验证者和用户均依赖 Hyperliquid 运营的中心化 API |
| **实际影响** | 2025.07 API 流量激增导致 37 分钟交易中断（链本身仍在出块） |

Hyperliquid 表示计划逐步开源代码和去中心化基础设施——当前架构反映的是"先快速迭代和安全加固，再逐步放权"的策略选择。

## 六、治理机制：经济激励下的无许可扩展

**Hyperliquid 改进提案（HIPs）** 通过经济门槛实现质量控制：

| HIP | 功能 | 机制 |
|-----|------|------|
| HIP-1 | 原生代币标准 | 31 小时荷兰拍卖获取 Ticker，市场驱动的部署成本 |
| HIP-2 | 自动基础流动性 | 新上线 HIP-1 代币自动获得 USDC 交易对基础流动性 |
| HIP-3 | 无许可永续市场 | 部署者需质押 500,000 HYPE，获得费用分成 |
| HIP-4 | 结果市场（期权/预测市场） | 已宣布，尚未上线 |

> 设计哲学：纯粹的无许可导致噪音和差体验；经济门槛确保认真的参与者，同时可能抑制实验性项目。这是生态质量与绝对开放性之间的有意取舍。

## 七、新兴竞争者

永续合约 DEX 赛道在 2026 年初已超过 20 个协议，月交易量合计突破 $1 万亿。

| 协议 | 定位 | 核心差异化 |
|------|------|-----------|
| **Lighter** | 安全优先 | 基于以太坊 L2，密码学证明验证每笔撮合和清算；零散户费用 |
| **Aster** | Binance 生态 | CZ 顾问，YZi Labs 投资；收益型抵押品（USDF ~17% APY）；1001x 杠杆 |

永续 DEX 持续从中心化交易所抢夺份额的结构性驱动力：
- 激进的激励计划吸引投机资本
- DEX 技术成熟度缩小了与 CEX 的执行差距
- 非托管安全性 + 代币奖励降低了转换成本
- Memecoin 投机资本正在向杠杆永续交易轮动

## 章节要点速览

1. **Hyperliquid 从零起步在两年内颠覆了永续 DEX 市场格局——dYdX 的份额从 75% 暴跌至 7%**
2. **核心飞轮：~99% 交易费 → HYPE 回购，将代币从治理工具变为现金流索赔权**
3. **HyperBFT 实现 0.2 秒中位终局 + 完全链上订单簿，此前被认为不可能**
4. **HyperCore + HyperEVM 双层架构在高速交易基础设施与 EVM 可编程性之间架起桥梁**
5. **HLP 在十月清算级联中盈利 $4,000 万+，证明了社区做市/清算金库的可行性**
6. **去中心化是最大短板：Foundation 控制 ~80% 质押，闭源软件，中心化 API 依赖**
7. **永续 DEX 赛道正结构性地从中心化交易所抢夺市场份额**
